1. Getting Started

1.1. Introduction

UB Data Migration Pro helps to port all your key data (Database Settings, Websites/Stores, Attributes, Categories, Products, Customers, Sales Data, Tax Data, Reviews & Ratings...) to Magento 2.

Since UB Data Migration Pro deals with direct database to database migration only, other perspectives like extensions, themes or customisation would be a separated task and would be best handled by the extension/theme providers accordingly.

UB Data Migration Support the following Magento versions:
  • Magento CE 1.x: 1.6.0.2 and later, 1.7.x, 1.8.x, 1.9.x
  • Magento CE 2.0.0 and later

1.2. Recommended Workflow

Following is our recommended workflow:

  • Step 1 - Install a fresh Magento 2 instance
  • Step 2 - Install UB Data Migration Pro extension
  • Step 3 - Proceed data mgiration and verify all migrated data
  • Step 4 - Once you complete all of the tasks in step #3, start with themes/extension integration.

2. How To Install

2.1. Pre-requisite:

  • Download Magento Community Edition 2.x from the link: https://www.magentocommerce.com/download
  • Before continue, make sure you always backup your Magento installation (files and database) so you can roll back in the event of issues.
  • Stop all activities in the Magento 1 Admin Panel and stop all Magento 1.x and Magento 2 cron jobs.
  • Assign Write permission to the pub folder at PATH_TO_YOUR_MAGENTO_2/pub

2.2. Installation Steps:

(For new installation)
  1. Copy the app folder and paste to the root folder of your fresh Magento 2 (Select the 'merge' folders/files option).
  2. Open your Terminal window, go to your Magento 2 folder and run the following commands to install this module:
    php -f bin/magento deploy:mode:set developer (optional)
    php -f bin/magento module:enable -c Ubertheme_Ubdatamigration
    php -f bin/magento setup:upgrade
    php -f bin/magento setup:di:compile (optional)

2.3. Enable PHP Memcached (Optional)

Enable PHP Memcached to improve the performance of the UB Data Migration Pro tool. Default settings of our tool uses cached file storeage. We highly recommend you enable the PHP Memcached to improve the overall performance.

  1. First, make sure you have the PHP memcached installed in your web server (Reference: https://www.digitalocean.com/community/tutorials/how-to-install-and-use-memcache-on-ubuntu-14-04)
  2. Once done, enable PHP Memcached to use with UB Data Migration Pro tool. Supposed you already install our migration tool successfully, simply open the configuration file at: PATH_TO_YOUR_MAGENTO_2/pub/ub-tool/protected/config/cache.php, then navigate to the lines: 6 -> 8 and:
    ++ comment out multiple lines from 6 -> 8
    ++ un-comment the lines from 11 -> 21

3. Upgrade Note

Please always backup your full Magento 2 site so you can roll back in the event of issues. We recommend to install our extension in a testing environment before deploying it to your production environment.

You simply follow steps below to upgrade to a newer UB Data Migration Pro version:

  • Step 1: Extract the latest version of our tool and upload to your Magento 2 web root folder as per Installation Guide in step 2.2 above.
  • Step 2: Run the command: php -f bin/magento setup:upgrade

4. Server Configuration

After installing UB Data Migration Pro, check the followings:

  • Step 1. Check your current Nginx configuration:

    If you have custom Nginx configuration, you need to check the following Nginx parameter: fastcgi_pass {your_value}
    - Open our UB Data Migration Pro's Nginx configuration file at: path_to_your_magento2/pub/ub-tool/nginx.conf.
    - Then find the line fastcgi_pass 127.0.0.1:9000;, and replace this parameter with your own value fastcgi_pass {your_value};.

  • Step 2: Open your Magento 2's nginx.conf file and add the following line to the end of that file: include path_to_your_magento2/pub/ub-tool/nginx.conf;
    Sample configuration in your nginx.conf file:
    server {
    listen 80;
    #...other configs

    #include config file for ub-tool
    include path_to_your_magento2/pub/ub-tool/nginx.conf;
    }
  • Step 3: Restart your Nginx server.

5. How To Use (Video Tutorial)

There are two ways to go about running data migration with our UB Data Migration Pro:

  • Running in GUI mode (under UB Data Migration Pro’s dashboard right in your Magento 2’s backend)
  • Running in CLI mode (see the section 5.2 How to proceed the migration using CLI below)

IMPORTANT: If you are migrating high volume of data into Magento 2, it is essential that you should proceed migration using the CLI mode after you complete all Pre-migration Setting steps.

5.1. How to proceed the migration:

This 9-minutes tutorial video will walk you through all installation and migration process, including delta migration steps: https://youtu.be/R4UBLe2kf-I

5.2. How to proceed the migration using Command-Line Interface (OPTIONAL)

Make sure you complete all Pre-migration Setting (8) steps first. Then open your terminal, navigate to your Magento 2 folder and run one of following commands. Once done, please follow the section 6 - Required Steps to complete migration process below.

Migrates Data All Steps:
php bin/ubdatamigration run

Migrates Single Step: Eg. The command to proceed migration in step #2 - Migrate Websites, Stores:
php bin/ubdatamigration run --step=2

Migrates Data Single Step with custom number of records per each batch runtime. Eg. The command to proceed the migration in step #5 with a custom 200 records:
php bin/ubdatamigration run --step=5 --limit=200

Resets Migrated Data All Steps:
php bin/ubdatamigration reset

Resets Migrated Data Single Step. Eg. The command to reset migration in only step #2 - Migrate Websites, Stores:
php bin/ubdatamigration reset --step=2

6. Required Steps To Complete Migration Process

NOTE: After you complete 8 steps in the above mentioned video tutorial, make sure you follow a few more steps (Required) to complete the migration process:

6.1. Copy media files:

  • Copy the folder at PATH_YOUR_MAGENTO_1\media\catalog and paste replace to PATH_YOUR_MAGENTO_2\pub\media\
  • Copy the folder at PATH_YOUR_MAGENTO_1\media\downloadable and paste replace to PATH_YOUR_MAGENTO_2\pub\media\
  • Make recursive write permission to "catalog" and "downloadable" folders which you have just copied.

6.2. Re-save each migrated Store View:

For each your migrated store, navigate to your backend, open each Store Views (as per this screenshot) and click Save button.

6.3. Clean Magento 2 cache:

In your terminal window, go to your Magento 2 folder and run the command: php -f bin/magento cache:clean

6.4. Re-Index the data:

In your terminal window, go to your Magento 2 folder and run the command: php -f bin/magento indexer:reindex

6.5. Re-Clean Magento 2 cache:

In your terminal window, go to your Magento 2 folder and run the command: php -f bin/magento cache:clean

NOTE: Since UB Data Migration Pro version 2.0.6, we apply a small change by appending the label of Attributes set / Attribute Group with the text 'Migrated'. You can edit with your own label.

7. Delta Migration

Delta migration is a completely new feature in UB Data Migration Pro V3.x. It enables to migrate incremental changes since the last time you migrate new data like Customers, Sales orders.... In short, it ports the following changes to Magento 2:

  • All newly created items in Magento 1 site
  • All changes to existing items in Magento 1 site which have been migrated to Magento 2 since the first migration.

Proceeding delta migration should be done using one of the Commands (CLI) below:

  1. Run delta migration for all steps: php -f bin/ubdatamigration run --mode=update

    IMPORTANT NOTE: the --mode=update will also overwrite existing Magento 2 data that has been migrated from Magento 1. This update mode will port the following data to Magento 2:
    - Newly created items
    - All existing Magento 1 items (including changed items) that have been migrated to Magento 2 since the first migration.

  2. Run delta migration for a specific step: php -f bin/ubdatamigration run --step=STEP_INDEX --mode=update
    For instance, you run delta migration for step #5 only, use this command: php -f bin/ubdatamigration run --step=5 --mode=update

How to migrate Customers changes:
  • Run the following command: php -f bin/ubdatamigration run --step=6 --mode=update
  • Once done, Reindex your data using the command: php -f bin/magento indexer:reindex
  • Then clean M2 cache with the command: php -f bin/magento cache:clean
How to migrate Sales Orders changes:
  • Run the following command: php -f bin/ubdatamigration run --step=7 --mode=update
  • Clean M2 cache: php -f bin/magento cache:clean

Apply similiar steps, for other data objects that you wish to migrate new incremental data changes. Please note, Reindex step is only needed if you migrate: Catalog Categories, Catalog Products, Customers data.

9. Troubleshooting & FAQs:

9.1. Attention - For Magento 2.1.0 only:

After you finish all migration steps as instructed above, if you see the following issues (which might occure in Magento 2.1.0 only), this is workaround to get it fixed:

If you can't edit the product in the back-end and see the issue as per this post.

HOW TO FIX:

  • Open the file: vendor/magento/module-configurable-product/Ui/DataProvider/Product/Form/Modifier/ConfigurablePrice.php
    and find the line:
    'config' => $visibilityConfig, and replace it with: 'config' => ['componentType'=>'container',$visibilityConfig],
  • Then clean the Magento cache by using command line: php -f bin/magento cache:clean

9.2. 404 Issues

If you run across 404 issues, the reason might vary. Check the following to see if it's your case:

404 issue might associate with the owner and file permissions at the /pub/ub-tool. You can try to set the folder to 755 and files to 644 respectively:

  • find . -type d -exec chmod 755 {} \;
  • find . -type f -exec chmod 644 {} \;

If you use Nginx Server, please make sure you follow the configuration steps mentioned in the Section #4. Server Configuration above.

9.3. Couldn't connected to Magento 1 database

If the source and destination databases are on different MySQL servers, you must enable a remote MySQL database connection to your Magento 1 database following this guide. 

It’s a better option in terms of migration performance if you connect directly to your databases in the same MySQL server rather than via a remote connection. 

9.4 . Enable debugging mode of UB Data Migration Pro

If you wish to view the issue log in more details while proceeding migration, you can enable debugging mode following steps below:

  • Open the file: PATH_TO_YOUR_MAGENTO_2/pub/ub-too/index.php
  • Find and uncomment the two lines below:
    #ini_set('display_errors', 1);
    and:
    #defined('YII_DEBUG') or define('YII_DEBUG', true);

9.5 Migration Logs

The migration tool generates the log file at: pub/ub-tool/protected/runtime/ub_data_migration.log which includes errors, warnings, and successes information.

You can also enable browser’s console to see any other issue that might occur during your migration process.
